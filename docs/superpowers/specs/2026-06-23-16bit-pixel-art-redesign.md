## 一、风格定义

### 1.1 核心定位

**16-bit 日式治愈像素风** — 参考星露谷物语、动物森友会、牧场物语等经典 JRPG/模拟经营游戏的视觉风格。

**关键词：** 温暖 · 治愈 · 田园 · 精致 · 成长感

### 1.2 风格特征

| 维度 | 描述 |
|------|------|
| 像素精度 | 32×32 基础单元，建筑最高 64×96 |
| 色数范围 | 单物体 16-32 色，整体色板约 128 色 |
| 线条处理 | 硬边像素，无抗锯齿，`image-rendering: pixelated` |
| 光影系统 | 左上方 45° 光源，顶/左/右三面明暗 |
| 色彩倾向 | 饱和但柔和，暖色调为主，绿色系丰富 |
| 细节密度 | 中等密度，每建筑有 5-8 个可辨识细节 |

### 1.3 与现有技术栈的关系

- **保留**：PixiJS 渲染引擎、等距地图系统、状态管理、UI 框架
- **替换**：Graphics 程序化绘制 → Sprite 纹理图集渲染
- **新增**：像素美术资产管线、SpriteSheet 打包、像素动画系统

## 三、建筑美术体系

### 3.1 五阶段成长视觉

沿用现有五阶段体系，但像素化重构：

| 阶段 | 等级范围 | 视觉主题 | 代表色 | 建筑风格 |
|------|---------|---------|--------|---------|
| 荒野 | Lv1 | 自然原始 | 森林绿 | 草地、树木、灌木、池塘 |
| 村落 | Lv2-3 | 田园木屋 | 暖木棕 | 木屋、猫窝、木栅栏、旧路灯 |
| 小镇 | Lv4-6 | 温馨小镇 | 暖米黄 | 联排屋、便利店、街灯、花园 |
| 城市 | Lv7-10 | 现代都市 | 冷蓝灰 | 公寓楼、办公楼、商场、红绿灯 |
| 都市 | Lv11+ | 繁华都市 | 深紫灰 | 摩天楼、地标塔、霓虹招牌、地铁 |

### 3.2 建筑清单（26+ 种）

#### 自然类（5种）
- 小树 / 大树 / 蘑菇 / 灌木丛 / 小池塘

#### 居住类（6种）
- 猫窝（村落） / 木屋（村落） / 联排屋（小镇） / 公寓楼（城市） / 住宅楼（城市） / 摩天楼（都市）

#### 商业类（5种）
- 小卖铺（村落） / 便利店（小镇） / 超市（小镇） / 办公楼（城市） / 商场（都市）

#### 设施类（5种）
- 木路灯（村落） / 街灯（小镇） / 红绿灯（城市） / 公交站（城市） / 地铁站（都市）

#### 装饰类（4种）
- 花圃（村落） / 公园长椅（小镇） / 广场喷泉（城市） / 雕塑（都市）

#### 地标类（2种）
- 钟楼（小镇） / 地标塔（都市）

### 3.3 像素绘制规范

#### 建筑三面画法（等距视角）
```
    /\    顶面（最亮）
   /  \
  /----\   左面（中等亮）
  |    |
  |----|   右面（最暗）
```

#### 标准光影比例
- 顶面：基础色 + 20% 亮
- 左面：基础色
- 右面：基础色 - 20% 暗
- 描边：基础色 - 40% 暗（1 像素）

#### 细节层次
每栋建筑至少包含：
1. 主体结构（墙面/屋顶）
2. 门窗（2-4 个）
3. 装饰细节（1-2 个，如烟囱、花盆、招牌）
4. 材质纹理（砖墙纹、木板纹、瓦片纹）

## 四、猫咪美术体系

### 4.1 三只猫咪像素化设计

#### 咪咪（橘猫）— playful 活泼
- 主色：橘色 `#ff8a65` + 深橘条纹 `#e64a19`
- 体型：中等，肌肉感
- 特征：背部三条深色条纹，白色下巴
- 表情：眼睛弯弯，笑口常开

#### 豆豆（灰猫）— lazy 慵懒
- 主色：灰色 `#9e9e9e` + 深灰条纹 `#616161`
- 体型：圆胖，肚子大
- 特征：圆脸，眯眯眼，粗尾巴
- 表情：半睁着眼，慵懒悠闲

#### 雪宝（白猫）— curious 好奇
- 主色：白色 `#fafafa` + 浅灰斑纹 `#eeeeee`
- 体型：修长，耳朵大
- 特征：蓝色眼睛，粉色鼻头，尾巴尖深色
- 表情：眼睛大睁，警觉好奇

### 4.2 动画帧设计

| 动作 | 帧数 | 说明 |
|------|------|------|
| idle（待机） | 2 帧 | 呼吸感，轻微上下浮动 |
| walk（行走） | 4 帧 | 四方向等距行走 |
| sit（坐下） | 2 帧 | 坐下 / 起身 |
| sleep（睡觉） | 2 帧 | 趴在地上，zzz 气泡 |
| play（玩耍） | 4 帧 | 扑打 / 追逐 |

### 4.3 像素尺寸
- 身体尺寸：约 24×20 像素
- 整体尺寸（含耳/尾）：约 32×28 像素
- 每只猫 5 个动作 × 4 方向 = 20 帧

## 六、技术实现方案

### 6.1 渲染管线改造

```
当前：PixiJS Graphics 程序化绘制
目标：PixiJS Sprite + Texture 图集渲染
```

#### 核心改动
1. **渲染器替换**：`tileRenderer` / `buildingRenderer` / `catRenderer` 从 Graphics API 改为 Sprite
2. **资源加载**：新增 `AssetManager` 统一管理纹理图集加载
3. **动画系统**：基于 `PIXI.AnimatedSprite` 的帧动画
4. **状态映射**：建筑等级 → 对应精灵图的映射表

### 6.2 资产组织结构

```
src/assets/sprites/
├── tiles/              # 地形瓦片
│   ├── grass.png
│   ├── dirt_road.png
│   ├── stone_road.png
│   └── water.png
├── buildings/          # 建筑精灵图
│   ├── nature.png      # 自然类（树/灌木/蘑菇）
│   ├── residence.png   # 居住类
│   ├── commercial.png  # 商业类
│   ├── facility.png    # 设施类
│   ├── decoration.png  # 装饰类
│   └── landmark.png    # 地标类
├── cats/               # 猫咪动画
│   ├── mimi.png        # 咪咪（橘猫）
│   ├── doudou.png      # 豆豆（灰猫）
│   └── xuebao.png      # 雪宝（白猫）
├── effects/            # 特效
│   ├── particles.png   # 粒子
│   └── glow.png        # 光晕
└── ui/                 # UI 图标
    └── icons.png
```

### 6.3 SpriteSheet 方案

- **打包工具**：使用 TexturePacker 或 free-tex-packer
- **格式**：JSON Hash 格式（PixiJS 原生支持）
- **命名规范**：`{类别}_{名称}_{等级}_{方向}_{帧号}`

### 6.4 生成策略（过渡方案）

考虑到初期可能没有专业像素画师，采用**渐进式策略**：

#### 阶段一：AI 生成 + 人工调整
- 使用 AI 像素生成工具（如 PixelMe、像素风格扩散模型）生成初稿
- 人工调整色板、修正等距角度、统一风格
- 产出可用的 MVP 版本

#### 阶段二：专业像素画师精修
- 邀请专业像素画师重绘关键建筑和猫咪
- 统一风格，提升品质
- 补充动画帧和细节

#### 阶段三：持续迭代
- 随版本迭代新增建筑和装饰
- 增加季节主题、节日主题皮肤

### 6.5 备选方案：代码生成像素图

如果完全不想依赖外部素材，可以考虑：
- 用 Canvas API 程序化生成像素图（而非 PixiJS Graphics）
- 预渲染到离屏 Canvas，作为纹理使用
- 优点：零素材依赖、风格统一
- 缺点：代码量大、调试困难、效果上限不如手绘

## 七、UI 适配方案

### 7.1 UI 风格调整

现有 UI 保留深色玻璃态风格，但增加像素元素：

| 元素 | 调整方案 |
|------|---------|
| 图标 | 像素化重绘（16×16 / 24×24） |
| 按钮 | 增加像素描边效果 |
| 字体 | 标题可考虑像素字体，正文保留现有 |
| 边框 | 关键面板增加 1px 像素风装饰边 |

### 7.2 像素字体候选
- **Zpix**（最像素）：中文像素字体，开源
- **Fusion Pixel**：融合像素，支持中英文
- 保留 Noto Sans SC 作为正文备用

## 八、动效与反馈

### 8.1 建筑放置动画
- 从上方掉落 + 弹跳 2 次
- 落地时尘土粒子效果
- 整体 0.5 秒完成

- 闪光效果（白色像素闪烁）

- 完成时彩色粒子爆开

### 8.3 猫咪互动动画
- 被点击时：跳跃 + 爱心粒子
- 喂食时：嘴部动画 + 满足表情
- 睡觉时：zzz 气泡浮动

### 8.4 环境动效
- 树叶轻微摇摆（2 帧切换）
- 水面波纹动画（4 帧循环）
- 云朵缓慢移动

## 十、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 像素美术资源产出慢 | 进度延期 | 先用 AI 生成 MVP，再人工精修；分阶段上线 |
| 等距角度不统一 | 视觉违和 | 制定严格的等距绘制规范，建立角度参考模板 |
| 移动端性能下降 | 体验差 | 控制同屏精灵数量；使用图集合批；LOD 分级 |
| 风格统一难度大 | 视觉杂乱 | 建立 Style Guide；指定一人把控整体风格 |
| 现有代码改造量大 | 重构风险 | 保留 Graphics 渲染作为 fallback；渐进式替换 |
